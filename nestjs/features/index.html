Ecosystem & Features NestJS

Beyond the core, NestJS ships first-party modules for almost everything you need in production. Naming these signals real-world Nest experience.

Data access

  • TypeORM (@nestjs/typeorm) — repositories, entities, migrations.
  • Prisma — type-safe query client (very popular pairing).
  • Mongoose (@nestjs/mongoose) — MongoDB schemas as injectable models.

Validation & config

  • ValidationPipe + class-validator/class-transformer — declarative DTO validation.
  • @nestjs/config — typed env config with ConfigService + schema validation.

Auth & security

  • @nestjs/passport + @nestjs/jwt — strategies (JWT, OAuth, local) wired as guards.
  • Guards + custom decorators (@Roles(), @CurrentUser()) for RBAC.
  • helmet, csurf, rate limiting (@nestjs/throttler).

API styles & transport

  • REST — controllers (default).
  • GraphQL (@nestjs/graphql) — code-first or schema-first resolvers.
  • Microservices (@nestjs/microservices) — TCP, Redis, NATS, Kafka, RabbitMQ, gRPC transports.
  • WebSockets (@nestjs/websockets) — gateways for real-time.

Productivity

  • Swagger/OpenAPI (@nestjs/swagger) — auto-generated API docs from decorators.
  • Testing (@nestjs/testing) — Test.createTestingModule with provider overrides for unit/e2e.
  • Scheduling (@nestjs/schedule), queues (@nestjs/bull), caching (@nestjs/cache-manager).
  • Lifecycle hooks — OnModuleInit, OnApplicationShutdown for setup/graceful shutdown.
  • Nest CLI — scaffolding (nest g resource users generates module+controller+service+DTOs+tests).
Why it scales: consistent structure + DI + first-party modules mean a new dev can navigate any Nest codebase, and features (auth, validation, docs, queues) are plug-in modules rather than bespoke glue.

🎯 Interview questions

How do you build a GraphQL API in Nest?

@nestjs/graphql with resolvers (code-first via decorators or schema-first), reusing the same providers/DI as REST.

How does Nest do microservices?

@nestjs/microservices with pluggable transporters (TCP/Redis/NATS/Kafka/gRPC) and message/event patterns (@MessagePattern, @EventPattern).

How do you unit test a service?

Test.createTestingModule({...}).overrideProvider(Dep).useValue(mock).compile(), then resolve and assert — DI makes mocking trivial.

How are API docs generated?

@nestjs/swagger reads controller/DTO decorators to produce an OpenAPI spec + Swagger UI automatically.
