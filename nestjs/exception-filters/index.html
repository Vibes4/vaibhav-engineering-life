Exception Filters NestJS

Exception filters are Nest's centralized error-handling layer — the equivalent of Express's 4-arg error middleware, but structured and type-aware.

Built-in HttpException

Throw an HttpException (or a subclass) from anywhere and Nest's default filter turns it into a proper HTTP response:

throw new NotFoundException('User not found');      // -> 404 { statusCode, message }
throw new BadRequestException('Invalid payload');    // -> 400
throw new UnauthorizedException();                   // -> 401
throw new HttpException('Custom', HttpStatus.IM_A_TEAPOT); // -> 418

Custom exception filter

@Catch(HttpException)              // which exceptions this filter handles
export class HttpErrorFilter implements ExceptionFilter {
  catch(exception: HttpException, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const res = ctx.getResponse();
    const status = exception.getStatus();
    res.status(status).json({
      statusCode: status,
      message: exception.message,
      timestamp: new Date().toISOString(),
    });
  }
}

Binding scope

  • @UseFilters(HttpErrorFilter) — method or controller scope.
  • app.useGlobalFilters(new AllExceptionsFilter()) — app-wide (the usual choice).
  • @Catch() with no args catches everything — for a global catch-all that also handles non-HTTP errors → generic 500.
Pattern: a global AllExceptionsFilter that maps known HttpExceptions to their status and unknown errors to a sanitized 500 — log the full error server-side, return a clean shape to the client.
Don't leak internals: never return stack traces/DB errors to clients. Log them; respond with a safe message + correlation id.

🎯 Interview questions

Express error middleware vs Nest exception filter?

Same goal (centralized error handling). Nest formalizes it: throw typed HttpExceptions, and @Catch() filters (method/controller/global) format responses with full DI and context.

What does @Catch() with no argument do?

Catches all exceptions (not just HttpException) — used for a global catch-all that maps unknown errors to 500.

How do you return a 404 in Nest?

Throw new NotFoundException('msg'); the built-in filter serializes it to a 404 JSON response.

Where do validation errors end up?

The ValidationPipe throws a BadRequestException (400) with the field errors, handled by the exception filter chain.
