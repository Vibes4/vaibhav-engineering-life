Controllers NestJS

Controllers handle incoming requests and return responses — the routing layer. Routes are declared declaratively with decorators instead of app.get(...) calls.

Routing decorators

@Controller('users')          // route prefix -> /users
export class UsersController {
  constructor(private readonly users: UsersService) {}

  @Get()                       // GET /users
  findAll(@Query('role') role: string) { return this.users.findAll(role); }

  @Get(':id')                  // GET /users/:id
  findOne(@Param('id') id: string) { return this.users.findOne(id); }

  @Post()                      // POST /users
  @HttpCode(201)
  create(@Body() dto: CreateUserDto) { return this.users.create(dto); }

  @Patch(':id')
  update(@Param('id') id: string, @Body() dto: UpdateUserDto) { ... }

  @Delete(':id')
  remove(@Param('id') id: string) { ... }
}

Parameter decorators

  • @Param('id') — route params (Express req.params).
  • @Query('q') — query string (req.query).
  • @Body() — parsed request body (req.body).
  • @Headers(), @Req(), @Res(), and custom decorators like @CurrentUser().
Return values become responses: just return an object/array and Nest serializes it to JSON with status 200 (201 for @Post). You rarely touch res directly — that keeps handlers testable.
DTOs: Define request shapes as DTO classes and validate them with class-validator + the global ValidationPipe. Controllers stay thin; validation is declarative.

🎯 Interview questions

How is routing different from Express?

Declarative decorators (@Get(':id')) on controller methods instead of imperative app.get(). Nest builds the route table from metadata.

How do you return a response?

Just return a value — Nest serializes it (200, or 201 for POST). Use @HttpCode() to override. Avoid @Res() unless you need raw control (it disables Nest's response handling).

What is a DTO?

A Data Transfer Object class describing the request payload, used with class-validator/class-transformer and the ValidationPipe for declarative validation + typing.
